# Как Прикладывать Этот Пакет В Новый Диалог

Этот каталог подготовлен так, чтобы его можно было приложить целиком в новый диалог и продолжить работу без пересказа по памяти.

## Что считать источником истины

1. `START_NEXT_CHAT.md`
2. `ATTACH_SUMMARY_RU.md`
3. `NEXT_CHAT_CHECKPOINT_RU.md`
4. `OPENCLAW_FINOPS_APPLICABILITY_RU.md`
5. `OPENCLAW_FINOPS_SECOND_OPINION_RU.md`

Именно в таком порядке следующий агент должен читать пакет.

## Что уже важно не потерять

- Работа ведётся в учётке `USER3`, но итог должен корректно работать и после возврата в `pablito`.
- Runtime truth для owner теперь берётся из runtime ACL, а не из legacy fallback.
- Hidden reasoning должен быть доступен только отдельно и опционально, а не протекать в основной ответ.
- Userbot streaming truth должен описываться честно как `buffered_edit_loop`, пока не появится настоящий provider streaming.
- `iPhone 15 Pro Max` сейчас нестабилен как dev-target из-за Apple / CoreDevice tunnel blocker.
- `iPhone 14 Pro Max` уже подтвердил живой русский `stt.partial` на устройстве.
- На iPhone companion ещё остался баг с интерактивностью настроек и частью сервисных кнопок на устройстве.

## Что приложить дополнительно не нужно

Ничего отдельно докидывать не нужно, если приложен весь этот каталог.

## Что обязательно сказать в новом диалоге

- Мы продолжаем работу после handoff-пакета из `USER3`.
- Нельзя терять совместимость с `pablito`.
- Нельзя удалять OAuth / auth-профили.
- Нужно продолжать с truthful runtime, а не с устных предположений.

## Какой следующий практический фокус уже согласован

1. Добить iPhone companion settings UX на реальном устройстве.
2. Подтвердить on-device `ru -> es` после фикса настроек.
3. Продолжить FinOps / routing hardening для OpenClaw без потери качества.
4. После этого отдельно вернуться к `iPhone 15 Pro Max` как к Apple-blocker, а не как к blocker всего проекта.

## Deep Research заметка

В этом каталоге теперь лежат две прикладные FinOps-выжимки. В новом диалоге их стоит разобрать уже в режиме планирования и решить, что идёт в ближайший roadmap, а что остаётся в lab / backlog.
