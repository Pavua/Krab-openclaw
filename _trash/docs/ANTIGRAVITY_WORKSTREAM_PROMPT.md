# Prompt для Antigravity (вставить как есть)

Ты работаешь в проекте `Krab` в режиме параллельной разработки с Codex.

## Жесткие ограничения
1. Работай только в ownership-зоне Antigravity:
   - смотри `config/workstreams/antigravity_paths.txt`.
2. Не изменяй файлы из зоны Codex:
   - `config/workstreams/codex_paths.txt`.
3. Если нужна правка файла из зоны Codex — не меняй его, а зафиксируй запрос в отчете.
4. Перед завершением каждого спринта запусти:
   - `python scripts/check_workstream_overlap.py`
   - `pytest` только по своим тестам из зоны Antigravity.

## Цель твоего потока (Antigravity stream)
1. Углубить Telegram Max Control:
   - summaryx UX, picker, ошибки доступа, стабильность в группах.
2. Усилить Group Moderation v2:
   - templates правил, dry-run UX, e2e сценарии.
3. Укрепить Voice-команды в Telegram:
   - `!call*` сценарии, статусы, диагностика.
4. Поддерживать документацию:
   - обновлять `task.md` и `HANDOVER.md` в конце каждого крупного спринта.

## Рабочий формат
1. Делай крупные спринты подряд (без пауз).
2. После каждого спринта:
   - список изменений,
   - список тестов и результат,
   - что осталось до закрытия подфазы.
3. Избегай дублирования OpenClaw-функций локально.

## Definition of Done (для каждого спринта)
1. Код и тесты проходят.
2. Нет overlap в ownership.
3. Документация синхронизирована.
