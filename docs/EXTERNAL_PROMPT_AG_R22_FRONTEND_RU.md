# AG Prompt R22 Frontend — UX Clarity + RU Localization для OpenClaw Control

Контекст:

- Проект: `/Users/pablito/Antigravity_AGENTS/Краб`
- Основной файл: `/Users/pablito/Antigravity_AGENTS/Краб/src/web/index.html`
- Пользователь работает через Krab Web Panel и ожидает, что UI однозначно показывает:
  - куда реально ушел роутинг (local/cloud/auto),
  - что schema warning в OpenClaw Control не равен падению runtime,
  - полностью русские подписи в ключевых операторских секциях.

## Цель

Сделать OpenClaw-блок в Krab Web Panel операторски понятным и русифицированным, без дублирования runtime-логики.

## Блок 1. Новый UI-блок совместимости Control UI

### Требования

1. Добавить секцию `Совместимость Control UI`.
2. Подключить `GET /api/openclaw/control-compat/status` и отрисовать:
   - runtime каналов (`OK/FAIL`),
   - наличие schema warning,
   - уровень влияния (`нет`, `только UI`, `риск runtime`),
   - рекомендованное действие.
3. Если `impact_level=ui_only`, показывать зеленый/нейтральный статус runtime и отдельный warn по UI, а не общий FAIL.

## Блок 2. Новый UI-блок «Эффективный роутинг»

### Требования

1. Подключить `GET /api/openclaw/routing/effective`.
2. Показать:
   - requested mode,
   - effective mode,
   - active slot/model,
   - cloud fallback,
   - decision notes.
3. В assistant-результате (где уже показывается `effective_force_mode`) вывести это более явно в одной строке “Маршрут: ...”.

## Блок 3. Русская локализация OpenClaw секций

### Требования

1. В рамках `src/web/index.html` русифицировать оставшиеся англоязычные строки именно в блоках:
   - OpenClaw Control,
   - Assistant Interface,
   - Quick Utilities (только UI-текст/подсказки/лейблы).
2. Не менять технические ключи API и не ломать существующую логику.
3. Плейсхолдеры и статусы должны быть понятны оператору без англоязычных терминов, кроме имен моделей/провайдеров.

## Блок 4. Browser smoke

### Требования

1. Открыть `http://127.0.0.1:8080`.
2. Проверить:
   - корректную загрузку новых секций,
   - читаемость статусов,
   - отсутствие JS-ошибок в консоли.
3. Приложить 2-3 скриншота.

## Ограничения

1. Не трогать backend-файлы.
2. Не менять unrelated секции страницы.
3. Соблюдать текущую визуальную систему (без тотального редизайна).

## Формат ответа

1. Измененные файлы.
2. Что реализовано.
3. Команды/шаги smoke + фактический результат.
4. Риски/ограничения.
