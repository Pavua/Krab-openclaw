# Ops/Monitoring Center UI Spec

## Концепт
Центр Мониторинга (Ops Center) предназначен для контроля за состоянием экосистемы Krab и OpenClaw. Интерфейс должен выглядеть как панель контроля миссий в Sci-Fi (Nano Banana Deep Space theme), с акцентом на "Здоровье" узлов и журнал событий (Journal).

## Состояния Интерфейса (Health States)
1. **Normal (Штатно)**:
   - Основной цвет: Зеленый (`--state-ok`, `#10b981`).
   - Иконки: Галочки, стабильные графики.
   - Бабблы / Бэйджи: Прозрачный зеленый фон с ярким бордером.
2. **Warn (Внимание/Деградация)**:
   - Основной цвет: Желтый/Янтарный (`--state-warn`, `#f59e0b`).
   - Иконки: Треугольники с предупреждением.
   - Триггеры: Например, Rate limits (429 от Signal), долгий ответ LLM, отключение fallback модели.
3. **Critical (Сбой/Критикал)**:
   - Основной цвет: Красный (`--state-bad`, `#ef4444`).
   - Иконки: Крестики, пульсирующие тревожные индикаторы.
   - Триггеры: Полный отказ базы данных, отказ OpenClaw Gateway.

## Макет Экрана (Layout)
- **Top Metrics (Health Board)**: 4-5 ключевых карточек с общим состоянием (Uptime, Memory, Active Sessions, DB Status).
- **Left Panel (Active Alerts)**: Список текущих открытых (Unacknowledged) алертов, требующих внимания, с кнопками Ack/Resolve.
- **Main Area (Journal Events)**: Непрерывный список логов аудита системы в виде таблицы или хронологической ленты.
  - Колонки: Время, Уровень (INFO/WARN/ERROR), Источник (KrabEar, OpenClaw, Router), Сообщение.
  - Возможность фильтрации по источнику или уровню.

## Взаимодействие (Interactions)
- Карточки Active Alerts меняют цвет фона в зависимости от состояния (`bg-surface` -> `bg-warn-transparent`).
- Кнопка **Acknowledge** переводит алерт в фоновый режим или скрывает его (цвет меняется на серый).
- Новые события в Journal Events плавно появляются сверху (slide-down) или лог автоскроллится (в зависимости от выбора порядка).
