# Интеграция с HomePod mini — Оценка возможности (Evaluation)

> **Статус:** Оценка завершена
> **Приоритет в бэклоге:** Низкий (требует высокой стабильности ядра Voice Gateway)

## 1. Концепция интеграции
Пользователь хочет общаться с Крабом в экосистеме Apple через Siri на базе HomePod mini. Домашние устройства должны иметь возможность:
1. Вызывать Краба через кастомные Apple Shortcuts ("Siri, ask Krab...").
2. Получать голосовой ответ (Audio Stream или TTS) напрямую из контекста Краба.
3. Управлять умным домом, если агент решит инициировать действие через HomeKit (например, включить свет).

## 2. Технические подходы

### Подход А: Apple Shortcuts (Быстрый путь)
- **Механизм:** Создать **Shortcut** "Краб", который делает HTTP POST запрос к нашему `Web API` (или напрямую к OpenClaw) с текстом распознанной речи (`Provided Input`).
- **Ответ:** Сервер Краба генерирует текст. Shortcut читает его с помощью встроенной функции `Speak Text`.
- **Плюсы:** Не требует написания нативных приложений (iOS/HomePod app). Работает из коробки на любом устройстве Apple.
- **Минусы:** Siri сама синтезирует речь (ответы не используют кастомные голоса Краба, например ElevenLabs/Azure-Dmitry).

### Подход Б: Audio Stream via Webhooks (Продвинутый путь)
- **Механизм:** Shortcut отправляет запрос, а сервер возвращает URL на аудиофайл или аудиопоток (сгенерированный TTS-движком Краба). Shortcut делает `Play Sound` по этому URL.
- **Плюсы:** Краб говорит своим голосом.
- **Минусы:** Возможны задержки (Latency) перед началом воспроизведения, так как HomePod будет скачивать/буферизировать аудио.

### Подход В: Официальная интеграция SiriKit / Matter
- Подходит для управления умным домом (HomeKit). Краб может получить доступ к устройствам через скрипты AppleScript/Homebridge, установленным на хостовом Mac, и управлять ими как "виртуальный ассистент".

## 3. Выявленные блокеры
1. **Стабильность Voice Gateway:** Для бесшовного опыта Time-to-First-Byte (TTFB) должен быть минимальным (менее 1.5-2 секунд). Текущие замеры TTS показывают, что облачные движки могут давать задержки.
2. **Apple Security Restrictions:** Запросы из Shortcuts к локальному IP (Macbook) могут требовать специальных разрешений или HTTPS, если телефон/HomePod не находятся в той же локальной сети (потребуется ngrok/Cloudflare Tunnel).

## 4. Рекомендация (Next Steps)
Сейчас интеграция **технически возможна** через подход A (Apple Shortcuts HTTP POST), но это отвлечет от фокуса на стабилизацию ядра.
**Вердикт:** Отложить реализацию до достижения 100% стабильности Telegram-транспорта и завершения рефакторинга Voice Gateway. Когда TTFB TTS стримера станет < 1 сек, реализовать подход Б.
