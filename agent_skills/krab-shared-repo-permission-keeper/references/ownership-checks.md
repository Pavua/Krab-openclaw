# Ownership Checks

## Что сначала проверить

- Какой именно файл или каталог не writable.
- Совпадает ли проблема у другой учётки на том же пути.
- Не лежит ли проблемный путь в `~/.openclaw`, `.env`, browser profile или другом account-local слое.

## Безопасные действия

- Исправить права только на одном repo-файле.
- Исправить права только на одном output-каталоге артефактов.
- Пересоздать конкретный generated-файл, если он не должен быть source-of-truth.

## Опасные действия

- `sudo chown -R ...` по всему дереву проекта.
- `chmod -R 777` как универсальная “починка”.
- Любые массовые git-reset операции ради permission issue.
