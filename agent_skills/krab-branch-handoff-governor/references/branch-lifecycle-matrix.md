# Branch Lifecycle Matrix

## Типовые сценарии

| Сценарий | Рекомендуемый ход | Что зафиксировать |
| --- | --- | --- |
| Один агент, одна задача | одна ветка `codex/...` | branch, scope, verification |
| Несколько агентов, disjoint write-scope | отдельные ветки или один branch с жёстким ownership | file ownership, merge-order |
| Helper-account готовит код, `pablito` делает финальный verdict | helper branch + финальная проверка на основной учётке | кто owner acceptance, какие evidence helper-only |
| Грязный worktree до начала работы | не чистить вслепую, сначала классифицировать | что было до старта, что относится к задаче |

## Что должно пережить handoff

- имя ветки;
- зона владения файлами;
- список фактически выполненных проверок;
- остаточные риски;
- что ещё нельзя трогать без синхронизации с другими агентами.
